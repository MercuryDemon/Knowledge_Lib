# Линтеры

# SYSCALL-ы

Системный вызов — это основной интерфейс взаимодействия между приложением в пользовательском пространстве (user space) и ядром Linux (kernel space)

примеры:
- read
- write
- open
- close
и т. д.


Сигналы — это механизм уведомления процессов о различных событиях в Linux. Это "софтверные прерывания", которые могут отправляться ядром, другими процессами или самим процессом себе.

примеры:
- SIGINT - ctrl + C , завершение с клавиатуры
- SIGQUIT- ctrl + \ , завершение + дамп
- SIGKILL - убить процесс (нельзя перехватить)
- SIGTERM - запрос на завершение процесса

Практические примеры использования сигналов: 

- Перезагрузка конфига - SIGHUP (nginx, sshd перечитывают конфиг)
- Graceful shutdown - SIGTERM (завершение текущих процессов и остановка)\
- Аварийная остановка - SIGKILL 
- Дамп стека - SIGQUIT (вызывеит паникуи показывает стек всех горутин)

# Примитивы синхронизации

## mutex
- Самый базовый примитив чтобы избежать одновременного чтения и записи в одну область памяти

## RWmutex (Read Write Mytex)
- Позволяет множеству горутин одновременно читать, но только одной писать.

## WaitGroup
- Ждем завершения всех воркеров

## Once
- Гарантирует, что функция выполнится только один раз, даже при вызове из множества горутин.

## Cond
- условная переменная, которая позволяет горутинам ждать наступления события.

## syncPOOL
- пул временных объектов. Позволяет переиспользовать объекты, уменьшая нагрузку на GC

## syncMap
- Потокобезопасная мапа

## Каналы
- Обмен данными между горутинами

## Atomic
- Для простых счетчиков и флагов можно использовать атомарные операции без блокировок.

# Оборачивание интерфеса в переменную 

Чтобы сделать функцию полиморфной - чтобы она могла принимать на вход любую переменную удовлетворяющую интерфейсу.

- Хранения значений разных типов в одной переменной
- Передачи в функции и возврата из функций
- Внедрения зависимостей (DI)
- Работы с неизвестными типами (JSON, пользовательский ввод)
- Type assertion для получения конкретного типа
