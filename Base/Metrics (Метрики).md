# Что это и зачем нужны.

Данные собираемые веб приложением для отслеживания активности пользователей и повышения эффективности использования сервиса.



# Подходы к реализации

## Push

- событие произошло, пишем данные в хранилище

- проще реализовать.

## Pull

- Микросервисы хранят в памяти компактные данные, потом некая сущность приходит их забирает например по HTTP. у каждого приложения есть отдельный эндпоинт с метриками.

- проще отлаживать отдельные приложения: не нужен досуп к общему хранилищу метрик, не нужно фильтровать данные по репликам.

# TSDB

Time Series Database - база данных временны́х рядов. Это специализированная БД, для работы с однотипными измерениями во времени. 

БД этого типа оптимизируют хранение какого-то числа, которое записано через равные интервалы времени. Проще понять на примере: чтобы собирать ежедневную температуру воздуха, нам надо хранить что-то вроде {(day1, t1), (day2, t2), ... } и больше ничего

- TSDB - хранят **время** и связанное с ним **число** и БОЛЬШЕ НИЧЕГО.


### Особенности TSDB

- реляционку умеют по минимуму: если SQL, то ограниченный, а то и вовсе его нет;
- типы хранимых данных урезаны;
- оптимизация на постоянную непрерывную запись;
- свой ~~упоротый~~ язык запросов
