# Stateful - система

- Хранит состояние у себя (в бд или кэше)

плюсы:
- просто реализовать

минусы:
- увеличение нагрузки на БД или кэш(много походов в бд для проверки состояния)\
- трудно масштабировать (один и тот же сервис запущеный одновременно на 2 серверах будет иметь 2 разных состояния, придется ходить в кэш ил БД для синхронизации)

Реализуется при помощи [[Sessions.  Сессии.]]

# Stateless - система

- Перекладывает ответственность хранения состояния на клиента.

плюсы:
- легко масштабировать
- меньше нагрузка на БД или кэш

Реализуется при помощи [[JWT token. Токены]]
