# Определение

По сути это фреймворк.
gRPC - google Remote Procedure Call - удаленный вызов процедуры(метода) какого либо сервиса с передачей в этот метод структуры данных в виде protobuf в качестве аргумента

- gRPC работате поверх протокола http/2, в качестве языка интерфейса используется - Protocol buffers.

почему http/2:
- передает данные в бинарном виде - это экономит место и увеличивает скорость пердачи сообщений
- Одно tcp соединение позволяет обрабатывать несколько запросов одновременно
- сжате заголовка -> меньше данных гоняем туда-сюда

преймущества **protobuf** в сравнении в JSON и XML:
- четкая схема данных со строгой типизацией
- меньший обьем данных
- поддерживается многими языками из коробки
- предоставляет средства обратной совместимости

gRPC - внутри себя автоматически генерирует код на нужном языке.
# Как это работает

1. Описываем в protobuf схеме api сервиса
2. генерируем код на нашем языке, поднимая gRPS server
3. сервис клиент по нашему api генерирует gRPC client на своём языке.

### Полезные плагины для расширения gRPC

- protoc-gen-grpc-gateway - позволяет работать с клиентами по REST api
- protoc-gen-openapiv2 - сгенерировать спецификацию в формате open api v2
- protoc-gen-go-vtproto - более быстрая сериализация данных
- protoc-gen-validate - позволяет валидировать сообщения автоматически
