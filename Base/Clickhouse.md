# Что это?

Clickhouse - распределенная аналитическая столбцовая(колоночная) СУБД

тоже реляционная как PostgreSQL только данные вносятся не в строки, а в колонки двумерной таблицы.

- использует только те колонки которые нужны в запросе, не нужно читать всю строку из базы, чтобы найти какоето определенное значение определенной колонки
![[Pasted image 20260317144754.png]]
- колонка - отдельная таблица с одной колонкой

![[Pasted image 20260317144918.png]]
- данные в колонках отсортированы и у каждого значения есть номер "строки" которой оно принадлежит
- колонки - отдельные файлы на диске
- использует синтаксис SQL, его диалект.


# Где используется

- Аналитика веб-проектов и мобильных проложений
- рекламные сети и real-time-bidding (торги в реальном времени)
- телекомуникации
- Электронная комерция и финансы
- Инфобез
- Бизнес-аналитика
- Онлайн- игры
- Интернет вещей


#  Особенности

- Масштабируется линейно
- Помимо интерфейсов CLI и TCP имеет HTTP - протоколдля отправки и получения данных
- нет PRIMARY KEY - вместо него показатель по которому происходит сортировка
- нет транзакция и блокировок
- принцип "когда-нибудь"
- сложность "точечной" работы, невозможжно или нецелесообразно находить и изменять одну конкретную строку
- высокие требования к оперативной памяти сервера
- возможность атак по HTTP

# Принцип "когда-нибудь"

СУБД не гарантирует, что данные придут в заданном порядке. Информация поступает в базу асинхронно и предсказать когда это случится - невозможно.

# Полезные практики

- не вставлять и не читать по 1 строке, вставлять и читаить сразу 10000, время для вставкиили чтения одинаковое для обоих случаев, если вставлять или читать по одной строке это займет в 10000 раз больше времени.
- агрегировать данные на стороне Clickhouse
- Подбирать правильные типы данных, чтобы более компактно сжимать базу.
