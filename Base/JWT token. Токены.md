# JWT

JWT - JSON Web Token.  Это ключ аутентификации пользователя. Используется для запросов к защищенным методам API. 

- Открытый стандарт для генерации токенов основан на форматк JSON
- использует симметричную(секрет) или ассиметричную(публичный/приватный ключ) подпись
- состоит из 3 частей: header, payload, signature

В токене ни в коем случае НЕЛЬЗЯ ПЕРЕДОВАТЬ конфиденциальную инфу: логин, пароль, контактнфые дынные.

В токене передаются:
- набор ролей определяющие права пользователя
- ID пользователя, который сервис может связать с учеткой.

Подпись создается алгоритмом который вычисляет определенным образом хэш из header и payload подмешивая к ним секретный ключ. (при изменении любой части JWT подпись изменится и не пройдет валидацию)

Секрет хранится на сервере, никто не должен его знать.

## Зачем нужны:
- для реализации stateless - систем. см. [[Stateful vs stateless wed-services design]]
- чтобы не передавать учетные данные пользователя с каждым запросом к серверу.

## Как это работает?

При аутентификации в сервисе клиенту возвращается JWT токен и хранится у него же, на всех серверах репликах, где развернут сервис, есть ключ от этого токена.

При дальшейшем взаимодейстии клиента с сервисом, он передает токен вместе с запросом.

Сервис его парсит и валидирует, если токен валидный, клиенту доступны защищенные методы сервиса, если не валидирует возвращается ошибка и предложение повторной аутентификации.

## Особенности

- Токены хранятся на стороне клиента, сервер ничего не сохраняет
- подписываются секретом (HMAC SHA256)
- Отправляются клиентом в заголовке Authorization
- Часто используются в  SPA, API для веб и мобильных приложений
- генерируется 2 пары: Access & Refresh
- Access token имеет малое время жизни (TTL минут 15 примерно)
- Refresh token нужен для обновления Accses token (TTL 30 дней)
- Часто хранятся в localStorage / sessionStorage. Они уязвимы к XSS атакам. Лучше всего хранить токены в HttpOnly cookie.


## Плюсы

- Позволяет реализовывать слабосвязную архитектуру и использовать как бля вэб интерфейсов так и для мобильных приложенийи т.д.

## Минусы

- При серьезной утечке токенов Серверу придется сменить секрет, следовательно все токены станут невалидными.
