# No SQL

No SQL - не реляционные базы данных. Все остальные БД которые не SQL таблички.

# Модели NoSQL DB

## Ключ - значение

У каждой записи в такой базе есть название поля и его значение.

пример: 
name: 'Никита' 
today : '05/12/2025'
president: 'Обэмэ'

- полезно для словарей и механизмов автозамены.


## Колонки 

Есть огромная таблица в которой содержаться все данные. Отличие от SQL в том, что SQL БД работают со строками, а NoSQL с колонками этой таблицы. Соответственно можно по одному запросу получить все данные из колонки.


## Графы

Представление БД в виде дерева. 

Дерево - данные хранятся по системе "родитель - отпрыски"
(типа как файловая система)

Графы - когда данные связаны хаотично ссылками друг на друга. (как у меня в Обсидиане)


## Документы

БД настольько умная, ято позволяет хранить данные в свободном формате.

## Хранилище данных временных рядов

Этот тип хранилища используется для наборов значений, которые обладают небольшим весом и привязаны к определённым моментам времени. Обычно это данные телеметрии, например датчика или датчиков интернета вещей.
![[Pasted image 20251205190255.png]]

## Хранилище объектов

Подобного типа хранилища используются для сохранения и извлечения данных большого веса, в качестве которых могут выступать видеофайлы, изображения, файлы приложений и т. д. Само хранилище обладает, как правило, также достаточно большим объёмом, что позволяет ему хранить подобное содержимое.

![[Pasted image 20251205190327.png]]


## Хранилище внешних индексов

Используется для индексированных данных других хранилищ, то есть подобный тип хранилища выступает в роли посредника, который ускоряет доступ к другим базам.


# Работа с запросами.

NoSQL БД не работают с SQL-запросами. У каждой такой БД есть своя методика работы с данными и свои инструменты реализующие CRUD (create, read, update, delete) функционал.


# Скорость и масштабируемость

NoSQL Бд можно легко разделить между несколькими компьютерами связанными в обшую сеть. Соответственно больше компов -> больше скорости и обьёма.


# Надёжность и безопасность.

NoSQL БД предлагают максимальную скорость работы, но решение всех конфликтов зависит от програсмиста. 
- НАДО ТЩАТЕЛЬНЕЕ СЛЕДИТЬ ЗА НАДЁЖНОСТЬЮ И КОНСИСТЕНТНОСТЬЮ!


# Применение

Все где большие объёмы данных и не требуется супер строгая целостность и надежность ( в плане недопущения утери данных)
